# feat/combat-stage — 敵ステージの作り直し

Deferred out of `feat/verdant-scenario` by user decision (2026-07-13): merge first,
then do this on its own branch. World-agnostic — it changes Default too.

## なぜやるか（実プレイで出た指摘）

現在の戦闘画面は**敵を二重に表現**している。3Dビューの中にスプライトを描き、その**下に
DOMのカード**（名前・HPバー・「残り2・健在」）を別途並べる。結果:

1. **カードが伸縮する** — 敵の数・状態文字列で幅と高さが変わり、レイアウト全体が揺れる。
   戦闘ログも行数で伸縮するため、1手ごとに画面がガタつく。
2. **スプライトはグループに1枚しか出ない** — ×2 でも絵は1体。数はカードの数字でしか
   分からず、絵と実態が食い違う。
3. **スプライトが小さく、迷宮から浮いている** — 床に接地しておらず、素材が活きない。

## 決定した形（user 承認済み）

- **敵カードは廃止**。敵は**絵だけ**で表す。
- **N体は N枚の絵**で描く（×2 なら2体並べる）。
- **各個体の足元に細いHPバー**、その下にグループ名。
- **レイアウトは固定高**。ログ行数・敵数・状態文字列で外枠が動かないこと。
- **スプライトを迷宮に接地させる**。床に立たせ、影を落とし、奥行きに応じて拡縮する。

## タスク

- **C1 スプライトの接地と寸法** — `dungeonScene.ts`: 敵ビルボードを床に接地、影、
  遠近スケール。1体あたりの見た目サイズを引き上げる。
- **C2 個体描画** — グループ1枚 → `count` 枚。隊列に沿って左右に散らす。
- **C3 カード廃止** — `CombatEnemyStage.tsx` の `.enemy-figure` DOM 行を削除。名前・HPバーは
  スプライト直下のオーバーレイへ。ターゲット選択のレティクル、被弾の数字もスプライト側へ。
- **C4 伸縮撲滅** — 敵ゾーンと戦闘ログを固定高に。ログは内部スクロール。
- **C5 テスト** — `combat-regression.spec.ts` を新DOMに追随。**レイアウト固定を数値で locked**
  （戦闘中に敵ゾーン／ログの bounding box が変化しないことを assert）。

## 決まっていること（追記）

- **味方ストリップのHP/MPバーは維持する**（user 確認済み）。削るのは**敵カード内のHPバー**だけで、
  それはカード廃止と同時に消える。敵のHPは各個体の足元の細いバーへ移る。

## 関連

  未納品の間は default の灰泥に落ちるため、森に灰のスライムが出る。C1–C3 はアセット到着と独立。
